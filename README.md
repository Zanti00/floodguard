# FloodGuard

FloodGuard is a Flutter application for monitoring flood conditions, managing connected sensor devices, and delivering safety alerts. It combines Supabase backend support, BLE device discovery, and mobile-first dashboards to help users stay informed about water levels and emergency contacts.

## Features

- Dashboard with water level readings and location selection
- Supabase-backed data retrieval for remote monitoring
- BLE device scanning and connection support via `flutter_blue_plus`
- Hotline and notification pages for emergency access
- Message and settings screens for user interaction and preferences
- Realtime water level visualization using chart widgets

## Installation

### Requirements

- Flutter SDK 3.9.2 or newer
- Dart SDK as included with Flutter
- Android Studio, Xcode, or web tooling depending on your target platforms

### Clone the repository

```bash
git clone https://github.com/Zanti00/floodguard.git
cd floodguard
```

### Install dependencies

```bash
flutter pub get
```

### Run the app

```bash
flutter run
```

To target a specific platform:

```bash
flutter run -d android
flutter run -d ios
flutter run -d chrome
```

## Project Structure

Only the main application directories are described below.

- `lib/`
  - `main.dart` - app entry point and Supabase initialization
  - `Controller/` - GetX controllers for device and app state
  - `pages/` - screen widgets for dashboard, messages, hotlines, settings, and more
  - `services/` - app services such as Bluetooth and OTP handling
  - `widgets/` - reusable UI components for charts and animated indicators
- `assets/data/` - sample data and JSON assets used by the app
- `android/`, `ios/`, `linux/`, `macos/`, `web/`, `windows/` - platform-specific files generated by Flutter

## Usage

After installation and configuration:

1. Launch the app on your chosen device or emulator.
2. Allow Bluetooth and location permissions if prompted.
3. Use the dashboard to view water level readings.
4. Navigate to Messages, Hotlines, and Settings via the bottom navigation bar.
5. Refresh data by pulling down on the dashboard.

## State Management

This project uses `GetX` for controller-based state management, especially for BLE scanning and reactive state updates.

- `lib/Controller/ble_controller.dart` manages BLE device discovery, scanning state, and result updates.
- Screen state is also managed locally in widgets using `StatefulWidget` and `setState` for UI updates.

GetX was chosen for its lightweight reactive model and easy dependency management in mobile workflows.

## API / Backend Integration

FloodGuard integrates with Supabase for backend data storage and retrieval.

### Data flow

- `main.dart` initializes Supabase using environment variables.
- `lib/pages/dashboard_page.dart` fetches water level records from a Supabase table named `water_levels`.

### Error handling

- Data loading errors are captured and displayed with retry support.
- Network exceptions are logged to the console.
